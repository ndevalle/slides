This is HTML

This is HTML (cont.)

Good HTML is only content. Presentation and interaction are separated layers in our code.

What about GREAT HTML?

What GREAT HTML is about

  1. Scalability

    Recode your styles and javascript. No need to touch your main code.

  2. Accessibility

    Allow readers and robots to understand your application just by looking at your skeleton.

  3. SEO (Search Engine Optimitation)

    Google will find you better if they can read your code properly.

  4. Performance

    The better your HTML is written, the less browsers will work to render it.

  5. Cross plattform applications

    Same code, every device.

IDs vs. Class

It's pretty simple:

  • IDs for element identification -> Javascript
  • Class for presentation -> CSS
<input type="text" class='security-code' id='securityCode'>

Why?

  • Less CSS code by recycling classes
  • Avoid mistakes with big development teams

    If a developer needs to change an ID for backend reasons, they don't need to think about the frontend impact of that change.
