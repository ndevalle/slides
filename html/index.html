HTML

What's HTML?

  • Stands for 'Hyper Text Markup Language'

  • Markup language; not programming language

  • Describes webpages using markup tags

  • Blah blah blah...

But for us...

HTML is this

Good HTML is only content. Presentation and interaction are separated layers in our code.

Why is it so important?

Why is it important?

  • Scalability and Maintenance

    Recode your styles and javascript. No need to touch your main code.

  • Accessibility

    Allow readers and robots to understand your application just by looking at your skeleton.

  • SEO (Search Engine Optimitation)

    Google will find you better if they can read your code properly.

  • Performance

    The better your HTML is written, the less browsers will work to render it.

  • Cross device applications

    Same code, every device.

But also...

Quality.

The <doctype>

  • Document type
  • Avoids Quirks Mode

The <head>

  • Title and metadata
  • CSS references

The <body>

  • All content
  • Javascript references

How to write good html

Separate content

HTML must be free of inline css or js. Content is what really matters.

<p style="color: grey; font-size: 11px;">Copyright 2011</p>
<p class="copyright">Copyright 2011</p>
<input type="button" onClick="">
<input type="button" id="confirmation">
		
$("#confirmation").click(){
	//do something
}

Less is more

Minimize the amount of html elements on your code. You'll help the DOM render it faster.

<div class="container">
  <div class="features">
    <ul>
      <li><span class="item">Pretty</span></li>
      <li><span class="item">Fast</span></li>
    </ul>
  </div>
</div>

vs.

<ul class="container features">
  <li>Pretty</li>
  <li>Fast</li>
</ul>

Do not lay out with <table>

Tables are for tabulated data ONLY.

Here some reasons:

  • They need more markup
  • They're semantically incorrect
  • It's impossible to redesign without refactoring your markup

In doubt? test it

IDs vs. Class

It's pretty simple:

  • IDs for element identification -> Javascript
  • Class for presentation -> CSS
<input type="text" class='security-code' id='securityCode'>

Why?

  • Less CSS code by recycling classes
  • Avoid mistakes with big development teams

    If a developer needs to change an ID for backend reasons, they don't need to think about the frontend impact of that change.

Other little tips

  • Don't forget the <doctype>.

    Avoid quiksmode!

  • Use external stylesheet and javascript files.

    One file each, even better!

  • Avoid "divitis". There's a bunch of elements you can use.

    <h1> <ol> <dt> <p>, etc

  • Do not resize images with html.

    Use 'width' and 'height' wisely.
