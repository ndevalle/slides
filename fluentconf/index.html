Your script just killed my site

A single point of failure (SPOF) is a part of a system that, if it fails, will stop the entire system from working.
Wikipedia.org

Your script just killed my site

  • Identify the SPOFs on your site: ads, widgets, etc
  • If you're building widgets that are going to be used on 3rd party websites, don't make them SPOF
  • <script src='script.js'> = SPOF
    Entire content blocked until they finish loading
  • Test your site and identify the weakest link
    Webpagetest.com to do so. Think about scripts blocked by firewalls (ex, China)

You Don't Need a Framework for that!

Stelle Weyl, Standardista.com

You don't need a framework for that!

Why do we write frameworks?

  • To normalize features across browsers
  • To be able to code easier

So... What's the problem about using frameworks?

  • We keep reinventing the wheel
  • Since they are faster to learn, there's almost NO reason to learn Javascript

... and that's scary

What's the problem about using frameworks?

We start doing things like this:

<script src="jquery.js"></script>

<script type="text/javascript">
  $('#content').addClass('touchable');
</script>

When we could have done this:

<script type="text/javascript">
  document.getElementById("content").classList.add("touchable");
</script>

Why do we care?

Mobile users care

  • Minimize Battery Usage
  • Reduce Latency (fewer HTTP requests)
  • Manage limited memory
  • Ensure a responsive UI

Parsing Javascript = consumes CPU & energy = Drains your user's battery

Why do we care?

Don't be a SPOF

Don't use frameworks on the 3rd party widgets you create!

Write Vainilla JS

  • If possible, make your homepage (or first page) work BEFORE the library needs to download
  • Your users are no-js users while the files are downloading, parsing and executing
  • Get and maintain mad JS skills!

Javascript is awesome but it's not always necessary!

This:

<script src="jquery.js"></script>
<script type="text/javascript">
  $('li:first').addClass('first');
</script>

Could be this:

<script type="text/javascript">
  document.querySelector("li").classList.add("first");
</script>

But it also could be this:

.li:first-of-type {
  /* .first declarations */
}

To sum it up...

  • If you can do it in a few lines of code, don't import a library
  • If you can make the home page work without requiring the library, do it!
  • Don't import library as part of 3rd party widgets.
  • Use libraries as tools, not as a crutch.
  • Learn to code JavaScript not just jQuery.
  • Remember mobile.
